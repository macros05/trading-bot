## Headline finding

**The live V7 config trades once in 24 months on BTC.** Across BTC+ETH+SOL combined it sums to 4 trades / 24 months. The V7 filter stack (MTF + volatility + session + range + short-trend) is so restrictive almost nothing fires. After 15 ablation variants on 3 symbols × 24-month walk-forward, **only one change improves multi-symbol Sharpe: disabling the volatility window filter.** Everything else either degrades multi-symbol performance or, in the case of short-side relaxations, blows up the account −80% to −99%.

Applied + deployed: `USE_VOLATILITY_FILTER = False`. All other live filters retained.

## Strategic implication

The V7 strategy class (RSI mean-reversion + MTF/ADX/session/range filters + asymmetric long/short) has been **exhaustively tested** at the parameter level. Best multi-symbol result is +0.57 Sharpe / 7 trades / 24 months / 3 symbols. This is at the **ceiling** of what filter-tuning can produce on top of this signal generator.

Marcos's stated goal — "muchas operaciones + máximo win rate + ganar rápido" — is **mathematically incompatible** with the current strategy class. Producing more trades requires either:
- **Looser filters** → catastrophic loss (proven by sweep).
- **More symbols** at current trade-per-symbol rate (8 symbols × ~3 trades/year = 24 trades/year). Still doesn't move the needle for a "bot perfecto".
- **New signal generators** (momentum breakout, funding carry, basis arb, pairs trading, microstructure features) — i.e. **the spec Fase 3+ multi-strategy ML pipeline**.

This sets up Session 2: start Fase 1 of the spec (TimescaleDB + ingestion infrastructure for the 8-symbol universe).
